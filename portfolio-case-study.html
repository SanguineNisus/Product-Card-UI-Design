Craigslist Case Study

A Case Study, Redesign, & User Feedback

Cumbersome Classified Cases

For a website that primarily operates on a local basis between its users to deal with various transactions, Craigslist comes off as cluttered, messy, and adds an unnecessary layer of hassling the users to achieve what they intend to do on Craigslist. The website being barebones and lackluster does not only detriment its usability, but also deters users away for more easily accessible options. A series of user testing has been done based on the problems posed by the current website:

 • Defining the problem
 • Having users test the current site
 • Alleviate user pain points
 • Ideating a more optimal experience
 • Prototype & validation

Once the user chooses a location that best suits where they would like to deal with their transaction locally, the user is met with another list of items, with a variety of topics ranging from community & services, buying & selling, classified job postings, and the like. There is an excessive amount of negative space that isn't utilized properly, and all the main items are set dead center of the page.


User Testing

With the assistance of five users to conduct testing and to point out user pains with the website, I have asked the users to utilize the website and explain their thought process as they go along working their way through the website:

 • Search for a local listing that has an item of interest. How was the search?
 • The user navigates the current listings. Elaborate on how information is displayed.
 • An ad/posting piques user interest. Does the ad/posting detail enough from the preview?
 • The user wants to specify filters to better fit their criteria. Was the user able to set filters accordingly and without issue?
 • The user is asked to search for multiple categories that interest them. Was there something that the user noticed between the categories?

After the users have spent time navigating through Craigslist to search for ads/postings of a variety of categories along with keeping in consideration on how they went about navigating, these issues tend to be the more prominent issues that reoccured often between all five users.

Users' Ideal Features

Ideating a Solution to User Pains

Having the results of the users testing out navigating through Craigslist, I have sufficient information to proceed and start ideating a solution that would address the issues that the users expressed, and that with certain aspects found on competing sites, core functions that make the user experience more enjoyable and hassle free would reflect for my solution to the user experience problem Craigslist has for these users. The Redesign of Craigslist

Utilizing the information I've gathered, having implemented most of the details in the low-fidelity wireframe, I've created a set of wireframes that also have basic prototyping features for the users to evaluate this new mockup site that has taken their concerns of the existing site and aimed to alleviate the issues.

Click here to view the prototype 

The users have also complied to testing out this prototype and have given feedback that reflects how this redesign looks, functions, and displays core elements and information. As a result, the feedback provided about this has been relatively positive, with a lot of components alleviating the issues most users have found, with this wireframe having:

 • Addressed the issue of endless lists the existing website has
 • Displayed images so that the user would be able to identify categories much more easily
 • A map that would be adjustable to easily pinpoint a specific location the user desires
 • Utilized more space to better make use of the components ( grids, scrolling buttons, circle avatars, etc. )
 • A method to easily set a post and navigate through past searches, categories, and previously made posting

What I've Learned & Personal Thoughts

Having the opportunity to redesign the Craigslist website has been an interesting experience for me, along with it being one of the earlier websites that I have made a wireframe and prototype of. Reflecting back on this, despite receiving postiive feedback, I ultimately felt that the wireframes could have been executed in a much more optimal manner, such as the display of certain objects within the wireframes, such as the pages on the signup registration, the login credentials retrieval page, and the post-going-live page. Was this a project that I would be glad to share without a second thought? Most definitely. We all start from somewhere, and take the experiences we've learned and improve ourselves to produce more sophisticated design in a sleek, minimalist, and ergonimic method possible. The main takeaway points that I have learned doing this project were:

 • Keeping form factors into consideration, such as space, margin & pixel sizes, & legibility
 • Analyzing what makes a brand have an identity ( and with Craigslist, it's the peace sign with Times New Roman font)
 • Considering the functionality factors of specific buttons that should be more thoroughly examined
 • Having select images be used to properly convey categories a lot more clearly
 • Keeping in mind the amount of minimizing clicks for the end user and implementing that into the case study.
